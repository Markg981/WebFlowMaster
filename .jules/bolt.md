## 2024-05-24 - N+1 Query in Test Plan Execution
**Learning:** The `runTestPlan` function in `server/test-execution-service.ts` was executing a separate DB query for each UI/API test inside the sequential execution loop, causing an N+1 query bottleneck.
**Action:** Always batch fetch related models using `inArray` and store them in O(1) memory lookup maps (e.g., `new Map()`) prior to looping when querying associations inside loops using Drizzle ORM.
## 2026-05-24 - Optimize Node.js Pagination with unionAll\n**Learning:** When retrieving and combining collections of models for pagination (e.g., UI and API tests), fetching everything into Node.js memory and performing array slicing/sorting is an O(N) performance bottleneck and memory sink.\n**Action:** Use Drizzle's `unionAll` to combine the queries directly within PostgreSQL/PGlite. This enables standard database-level pagination using `.limit()` and `.offset()`, massively reducing query latency and backend memory consumption.
