## 2024-03-27 - [Playwright Browser Pool Leak]
**Learning:** [When converting manual browser launches (`browserEngine.launch()`) to utilize a shared pool via `browserPool.acquire()`, it is critical to explicitly call `await context.close()` and `await page.close()` in `finally` blocks before executing `await browserPool.release(browser)`. Unlike closing the entire browser process, which terminates all attached child resources automatically, returning a browser to the pool leaves the context and page active, resulting in a severe memory leak as contexts accumulate in the long-running browser instance.]
**Action:** [Next time I integrate a pooled browser implementation in Playwright or Puppeteer, I will meticulously review every method referencing the pool to ensure that contexts and pages are explicitly destroyed during cleanup, even on success paths or uncaught exceptions.]
