## 2024-05-24 - N+1 Query in Test Plan Execution
**Learning:** The `runTestPlan` function in `server/test-execution-service.ts` was executing a separate DB query for each UI/API test inside the sequential execution loop, causing an N+1 query bottleneck.
**Action:** Always batch fetch related models using `inArray` and store them in O(1) memory lookup maps (e.g., `new Map()`) prior to looping when querying associations inside loops using Drizzle ORM.
## 2026-07-01 - Optimizing Drizzle Iterations and Cleaning PRs
**Learning:** Multiple array iteration loops combined with filters/some can easily mask O(K*N) complexity. Furthermore, during test execution, auto-generated files like migration journals and workspace modifications (e.g., package.json scripts tweaks) may creep into source control. Using `git clean` or explicitly restoring `package.json` guarantees that the agent strictly adheres to negative constraints.
**Action:** When calculating statuses over large detailed DB result sets, avoid chained `.filter()` and replace them with a single `for...of`. Before creating PRs, verify `git status` explicitly to ensure no lockfiles, test DBs, or temporary configuration tweaks are included in the staging area.
