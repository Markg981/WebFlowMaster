## 2024-05-24 - N+1 Query in Test Plan Execution
**Learning:** The `runTestPlan` function in `server/test-execution-service.ts` was executing a separate DB query for each UI/API test inside the sequential execution loop, causing an N+1 query bottleneck.
**Action:** Always batch fetch related models using `inArray` and store them in O(1) memory lookup maps (e.g., `new Map()`) prior to looping when querying associations inside loops using Drizzle ORM.
## 2026-06-10 - O(N) Memory Bottleneck in Selectable Tests\n**Learning:** The `GET /api/selectable-tests` endpoint was fetching entire tables into Node.js memory, combining, sorting, and slicing them for pagination. This creates an O(N) memory bottleneck that scales poorly.\n**Action:** Use Drizzle ORM's `unionAll` to combine unawaited subqueries directly at the database level. Push `.orderBy()`, `.limit()`, and `.offset()` down to the database to perform pagination natively without loading unneeded rows into memory.
